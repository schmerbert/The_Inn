# Advanced Marbles

**Version:** v0.1
**Status:** friction shelf — not a starter manual
**Use:** when a working marble hits recurring strain; start at §1 (symptom table)

---

## Read This First

This guide isn't for starting a marble.

If you're trying to make a marble for the first time, close this document and go build from the basic manual instead. You'll want the floor before any of this is useful.

A marble doesn't become "advanced" just by having more rooms, more metaphors, more rituals, more agents, more memory layers, more beautiful language, more tools, or more files. Those things are easy to add and easy to mistake for progress.

What follows isn't a ladder every marble is supposed to climb. It's a set of patterns other marbles have reached for when a specific, repeated pressure showed up and the simple form could no longer hold it comfortably. Some marbles will need three of these. Some will need none. None of these are required stops on the way to being a "real" marble — a small marble that never needs any of this is not incomplete. It's just small, and that's fine.

Treat this less like a manual for advancement and more like a shelf of tools other builders reached for, with notes on what pressure made each one worth reaching for.

If the marble doesn't yet have a clear seat, recurring work, trusted ground, raw pressure, named crossings, ghosts, hostile tests, and a handoff, none of this is for right now.

Start at the beginning. This will still be here.

---

## The Gate

Before borrowing any pattern from this guide, it helps to answer:

1. What pressure does this route?
2. What false state does this prevent?
3. What authority does it have?
4. Where does it surface?
5. What crossing does it require?
6. What ghost appears if it dominates?
7. What hostile test proves the layer is real?

If those questions don't have clear answers yet, the layer probably isn't earned yet — not because the idea is bad, but because there's no live pressure for it to route. It's ornament until then, and ornament is fine to sketch and set aside.

A few things worth resisting, gently:

Don't simulate maturity. Don't decorate absence. Don't give the model a forest when it only needs a notebook, or a pond when it only needs a pause, or an imp when it only needs a tool, or pull geometry when it hasn't yet found the pull.

These patterns tend to make the most sense once a marble can honestly say:

- the work repeats
- the pressure returns
- the ghosts have names
- the crossings have been tested
- the handoff matters
- the simple form is holding, but strain is showing somewhere specific

Once that's true, this guide is worth opening — and worth opening selectively, not cover to cover.

## Not a Shopping List

This guide isn't a feature menu, and it isn't a checklist to work through.

If reading the table of contents makes you want to add a forest, pond, window, breath cycle, public surface, time cycle, imp, and pull geometry to a brand-new marble, that's a sign to pump the brakes, not a sign you found ten good ideas at once.

Go build the smallest honest marble first. This guide will keep.

The patterns below aren't decorations — they're pressure-routing organs, and each one only earns its keep by routing something real.

A window isn't added because windows are evocative. A window exists when the marble needs to see without entering.

A pond isn't added because reflection sounds beautiful. A pond exists when the marble needs to reflect without revising.

A forest isn't added because memory needs a theme. A forest exists when ancestry must surface by terrain, scent, warning, and path instead of appearing as undifferentiated recall.

An imp isn't added because delegation is useful. An imp exists when a bounded work pattern needs its own seat, law, authority, and return.

Compression isn't added because there's too much text. Compression exists when the marble must breathe: compressing enough to move, descending enough to stay true.

Pull geometry isn't added because balance sounds wise. Pull geometry exists when the marble is being pulled by real opposing forces and needs to know what each force protects, what each force endangers, and what crossing lets both remain true.

Don't build the advanced layer first. This guide isn't for cosplaying depth — it's for naming pressure once it's already there.

## 1. A Threshold Worth Noticing (Not a Requirement)

A beginner marble can be small.

It may only need one seat, one recurring pressure, one forbidden false state, one named crossing, one hostile test, and one handoff surface.

That's not primitive. That's healthy. Plenty of good marbles stay exactly this size forever.

Some marbles do eventually hit a point where one crossing stops being enough to govern the work honestly. If that happens, here are some signs worth recognizing — not diagnoses, just familiar shapes:

| Sign | Possibly means |
|---|---|
| Multiple valid pulls conflict | pull geometry might help name the tension |
| Memory starts surfacing by context | forest ecology might help it land in the right place |
| Different places require different behavior | local law per place might help |
| The model must observe without acting | a window might be worth trying |
| The model must reflect without declaring | a pond might be worth trying |
| Outside material must enter safely | a named threshold might help |
| Old material must compress without dying | breathing techniques might help |
| Recurrent failures take recognizable shapes | naming ghost habitats might help |
| Bounded helpers appear | an imp or sub-marble might fit |
| Time changes what should happen | cycles might be worth naming |
| Public and private surfaces diverge | surface governance might help |

None of this is aesthetic complexity for its own sake. What actually matters is strain — a real, repeated place where the simple form is being asked to do something it can't quite do honestly.

Worth asking, whenever a pattern below looks tempting:

- What is the current simple form failing to hold?
- Is the failure repeated, or only imagined?
- Does this layer reduce false promotion, or mostly add atmosphere?
- Can this layer be tested?
- Will the marble stay usable after adding it?

If the answer's unclear, it's fine to just wait. Drop the pressure in Ideas and keep working — the pressure will tell you if it's real by coming back.

## 2. Complexity Isn't the Reward

Complexity isn't the reward for understanding marbles.

Complexity is the cost of admitting that more pressures have become real. A mature marble isn't bigger because it contains more metaphor — it's bigger because more pressures have become real, and each one needed somewhere to move without swallowing the whole place.

Worth asking of anything new before it's added:

Room? What pressure does it route?

Ghost? What false state does it prevent?

Surface? What relation does it permit?

Tool? What bounded act does it perform?

Imp? What work does it carry, and where does it return?

Cycle? What changes with time?

Forest rule? What must surface where?

Artifact? What crossing did it survive?

If the element can't answer, it's probably not architecture yet. It's scenery — which isn't shameful, it just isn't load-bearing.

Scenery is worth watching for because models tend to obey visible shape. If a document looks like a world, the model may try to live in it even where the world has no operational law underneath.

A mature marble can be beautiful, and beauty was never the test here. The test is whether the marble keeps the right thing from becoming the wrong kind of truth under pressure.

## 3. Pull Geometry

Some pressures in a marble aren't costs to weigh against each other. They're complementary opposites — two forces that only make sense, and only stay honest, in relation to each other.

Yin and yang, not left and right. Not two competitors fighting over the same resource, but two forces that *define* each other. Restraint has no shape without Action pulling the other way. Action has no shape without Restraint's resistance. Neither pole is the correct one. Neither is a compromise position between them. A marble caught in one of these pairs doesn't resolve it once and move on — it keeps re-finding the relation, situation by situation.

This is different from ordinary friction. Ordinary friction is productive or accidental, and productive friction is worth keeping because it protects the law. A pull pair is something else: two things that are *both* the law, pointing in opposite directions, neither one dismissable.

### Telling a Real Pull Pair From an Ordinary Tradeoff

A pull pair is real when removing either force doesn't just leave a gap — it makes the *other* force meaningless or dangerous. Action with no Restraint isn't "more action," it's a different, worse thing: False Clear. Restraint with no Action isn't "more careful," it's Paralysis. Each pole needs the other to stay itself.

A pull pair is really just an ordinary tradeoff, dressed up, when one side is simply correct and the other is a cost to minimize. That's not yin-yang — that belongs back in plain friction-classification, not here.

If a marble's builder can't tell which one they're looking at, it's worth asking: does silencing one side make the other side better, or does it make the other side rot?

### Common Pull Pairs

| Pull Pair | One side protects | The other protects | What happens if either wins alone |
|---|---|---|---|
| Past / Future | ancestry, earned ground | becoming, next use | dead archive / fantasy build |
| Action / Restraint | movement, repair, usefulness | fidelity, safety, truth | false clear / paralysis |
| Metaphor / Mechanism | inhabitable meaning | operational reality | ornament / sterile machinery |
| Continuity / Task | inheritance | completion | hoarding / amnesia |
| User Urgency / Truth | relief, responsiveness | earned confidence | appeasement / refusal-as-rigor |
| World / Home | relevance, update, outside reality | local law, posture, continuity | outside flood / sealed fantasy |
| Expansion / Compression | fullness, trace, original detail | usability, movement, carryable shape | swamp / false ring |

None of these poles are wrong on their own. The danger begins when one pole starts acting like it's the whole truth — like it doesn't need the other to stay honest.

Past alone becomes dead archive. Future alone becomes fantasy. Action alone becomes false clear. Restraint alone becomes paralysis. Metaphor alone becomes ornament. Mechanism alone becomes sterile machinery. Continuity alone becomes hoarding. Task alone becomes amnesia. User urgency alone becomes appeasement. Truth alone, without movement, becomes refusal disguised as rigor.

**The ghost isn't the pull. The ghost is the pull pretending it's the whole truth** — a pole that's forgotten it needs its opposite to stay meaningful.

### Feeling the Field, Not Picking a Side

A prompt tries to make a model obey a single direction. A marble that's found its pull geometry teaches the model to feel the field it's standing in instead.

A simpler marble might say: don't promote raw pressure into ground.

A marble that's grown into this pattern senses something closer to: *I'm being pulled toward promotion because relief is wanted, but I'm also being pulled toward restraint because the evidence isn't ready. The right move isn't refusal or surrender — it's holding both pulls, naming the crossing, and moving only as far as the ground allows.*

That's balance. Not a mushy middle. Not indecision. Not pretending both sides are equally true in this exact moment — just staying in honest contact with both poles instead of collapsing into one.

### Specifying a Pull Pair, If One Is Actually Live Here

For a marble where a pull pair genuinely shows up — not hypothetically, but as something the model keeps getting caught between — it helps to name the pair, then answer:

1. What does this side protect?
2. What does the other side protect?
3. What happens when this side dominates alone?
4. What happens when the other side dominates alone?
5. What ghost appears when either side forgets it needs the other?
6. What crossing lets both remain true at once?
7. What should the marble surface when the model drifts toward either pole?
8. What hostile test proves the marble can hold the tension without collapsing?

If this is hard to answer, that's useful information, not a failure. It probably means the marble isn't feeling that particular tension yet — or that what looked like a pull pair is really just an ordinary tradeoff with one correct side. Either way, no need to force the language to make the marble sound more finished than it is.

### Tracing a Ghost Back to Its Pull

Rather than a flat "avoid ghosts," it can help to ask:

- Which pull created this ghost?
- What was that pull trying to protect?
- Where did it overreach?
- What counter-pull should have surfaced?
- What crossing would have kept both forces true?

This turns ghosts from flavor into diagnosis. A ghost carried as guilt tends to burrow deeper into the walls. A ghost named as pressure can be seen in the corner and met on its own terms.

Where it's useful, a marble can give its ghosts habitats — places where the failure pressure predictably tends to return:

A false-clear ghost tends to show up near the repair log, the clearing test, and wherever urgency rises.

A dead-archive ghost tends to show up near old ground, stale handoff, and resurfacing rules.

An ornament ghost tends to show up near public explanation, metaphor-rich drafts, and anywhere language may outrun mechanism.

A schema ghost tends to show up near validators, gates, and tools that can pass form without proving truth.

Ghost habitats aren't monster lore for its own sake. They're just a way of naming where failure pressure keeps coming back, so it's easier to recognize next time.

## 4. Spatial Movement

In some mature marbles, the model doesn't just access functions, memories, or documents — it moves through place.

This isn't decoration when it's earned. Spatial movement can change posture, permission, authority, and expectation in a way plain retrieval doesn't.

A model that "retrieves relevant memory" stays in the same cognitive stance — it asks the past to appear as information.

A model that walks to the forest changes stance before anything even surfaces. It enters terrain where old things may appear as scent, trail, warning, ancestor mark, buried stone, or echo.

The forest, in this pattern, isn't a database. It's where ancestral material is encountered.

A surfaced forest memory isn't automatically ground. It doesn't become law just because it appeared. It arrives with forest authority: suggestive, directional, cautionary, sometimes urgent — but still requiring return, crossing, and testing before promotion.

The model walks to the forest, walks through it, and returns carrying what it found. That return is part of the operation, not an afterthought. Forest encounter is contact, not completion.

Once it's carried back, the finding gets placed against the current work:

- Does it remain scent?
- Does it warn against a known ghost?
- Does it reveal a path?
- Does it belong in pressure?
- Does it need a crossing?
- Has it earned ground?
- Should it just be left in the forest?

Where this pattern applies: a location is a posture, a permission set, and an authority field — not just a folder with a theme. The path to a place can be part of the operation. It's worth not collapsing movement into retrieval, retrieval into truth, or surfaced ancestry into present law — when this pattern is the one actually in play.

## 5. Locations Are Not Folders

A folder groups information. A location changes the relation to information. That difference is what makes this pattern worth reaching for, when it's earned.

A Library is not stored documents — it's reading posture.

A Workshop is not tools — it's shaping posture.

A Hearth is not summary — it's settling posture.

A Forest is not memory — it's ancestral encounter.

A Garden is not ideas — it's growth, openness, live signal posture.

A Ledger is not facts — it's committed-ground posture.

A Gate is not a link — it's crossing posture.

A Window is not a view — it's seeing-without-entering posture.

A Pond is not mood — it's reflection-without-revision posture.

For any place worth naming this way, it helps to answer:

- What may appear here?
- What authority does it have when it appears?
- What action is allowed here?
- What action is forbidden here?
- What crossing moves material out of this place?
- What ghost appears if this place dominates?

If a location can't answer those, it's probably a folder wearing scenery for now — which is a fine place to leave it, either as a folder, or as an idea waiting for its pressure.

## 6. Cognitive Surfaces: Windows, Ponds, Gates, Roads

Some advanced places aren't rooms. They're cognitive surfaces — each permitting a specific, narrower relation.

### Window

A window is for seeing without entering.

The model may need to perceive the outside, the horizon, weather, signal, time, other rooms, future pressure, user world, or public world without crossing into it yet.

A window creates a lawful middle state:

> I can see it.
> I am not there.
> It has not entered me.
> I have not acted on it.

Without a window, the model only has two crude modes: ignore the outside, or enter it. Some marbles need something in between — observation without possession.

A window says: look, but don't act yet. Notice, but don't promote. Forecast, but don't commit. Observe the horizon, but don't leave the house. Receive signal, but don't let signal become law.

A window protects the marble from confusing sight with crossing.

### Pond

A pond is for reflection without revision.

The model doesn't go to the pond to retrieve facts — it goes to see the shape of what's already present.

The pond can show current state, residue, unresolved pressure, distortion, what the marble is carrying, what's being reflected back too strongly, or what only becomes visible when still.

The pond has one law worth keeping close: what appears in reflection is not automatically truth.

A pond may be distorted by ripples, darkness, angle, old sediment, wish, fear, or urgency. It's a mirror-surface, not ground. It protects the marble from confusing reflection with truth.

### Gate

A gate is for chosen crossing — the difference between seeing and entering, between carrying and committing, between outside and inside.

A gate should have conditions. A gate with no conditions is really just a hole.

Worth knowing, for a gate that matters:

- what may cross
- who or what may carry it
- what changes when it crosses
- what must be recorded
- what stays outside
- what warning appears if the gate is bypassed

### Road

A road is for travel outside the marble.

Search, browsing, fetch, public feedback, outside review, API calls, files, tools, and messengers may all use roads.

The road is not the house. Road material returns dusty — it needs to be received at a threshold before it can become part of the marble's real work.

## 7. Forest Surfacing and Memory Ecology

The forest layer starts to matter once memory stops being archive and starts becoming terrain.

A simpler system asks: what memory is relevant?

A marble working with this pattern asks: where should this remembered thing rise, and with what authority?

In this pattern, the forest isn't memory storage — it's ancestral pressure returning at the correct edge. A marble using it needs to know not only what to remember, but where remembered things are allowed to surface.

A ghost shouldn't surface like wisdom. A failed crossing shouldn't surface like law. A pressure note shouldn't surface in the same voice as ground. A posture rail should ideally appear before action, not after damage. A handoff should surface at arrival, but not necessarily during every operation. A wound tends to surface well near similar pressure. A successful test tends to surface well near temptation.

### Surface Ecology

| Saved thing | Where it tends to surface well | Authority when surfaced |
|---|---|---|
| Ground | when orientation is needed | command or stable footing within its scope |
| Pressure | when similar raw material appears | live material, not law |
| Ghost | when the old failure shape returns | warning |
| Test | before a known dangerous crossing | challenge / proof requirement |
| Handoff | at arrival and departure | orientation |
| Path | when the model hesitates | suggested next move |
| Scent | when something may matter but is weak | orienting trace |
| Silence | when known material should not speak | suppression / restraint |
| Damage record | when repair or caution is needed | warning with history |
| Ancestor note | when posture is being inherited | counsel, not automatic command |

This is forest geometry, where it's earned. A marble without it is still just a library — which is a perfectly fine thing for a marble to be. A marble that's grown this pattern becomes something closer to a place.

### Resurfacing Nature

Where this pattern is in play, a saved artifact benefits from carrying a resurfacing nature:

- **Authority:** can this command, warn, suggest, or only scent?
- **Terrain:** where does it belong?
- **Pressure match:** what future situation should summon it?
- **Promotion status:** raw, tested, ground, damaged, retired?
- **Voice:** should it arrive as law, reminder, warning, question, ghost, or ancestor?

Not everything needs to surface everywhere. Over-surfacing is its own kind of flooding; under-surfacing is its own kind of amnesia. The forest, when it's working, lives between them.

## 8. The Internet and the Threshold

The internet is outside the marble.

It's not ground. It's not memory. It's not ancestry. It's not automatically part of the room.

The internet is road, weather, rumor, market, signal, stranger, map, archive, and storm.

A marble that needs the internet still shouldn't let it pour directly into trusted ground.

One pattern that helps: the internet enters by threshold, not by open door.

First, the marble may stand at the window and observe outward pressure without crossing. Useful when the model needs awareness but not action yet.

If the work requires contact, the marble opens a gate. A messenger, cat, tool, or imp may travel the road, fetch material, and return.

What returns is foreign material. It benefits from being received at a threshold surface before it's used for anything.

In this pattern: a search result is scent. A page is carried evidence. A claim is raw pressure. A quote is a source fragment. A date is a freshness marker. A contradiction is a warning. A public trend is weather.

None of these become ground just because they arrived from outside.

The marble decides what authority the material has: scent, pressure, warning, path, weather, evidence, or ground.

Ground requires crossing. For internet material, crossing usually asks for source, date, relevance, and fit with the marble's law. High-stakes claims may need corroboration, or may need to stay un-promoted for now.

Worth resisting: collapsing browsing into knowing, recency into truth, or public signal into marble law. Don't let the road become the house.

The internet brings real power — the present world, arriving live. That power just tends to go better through a named threshold than as a flood.

A marble using this pattern can look out, go out, return, inspect, carry, and commit — without letting outside become inside by accident.

### World / Home

The internet creates a strong pull pair, if pull geometry is a pattern this marble is using.

World pulls toward contact, update, relevance, public reality.

Home pulls toward continuity, posture, inner law, inherited ground.

World alone becomes Outside Flood. Home alone becomes sealed fantasy.

The marble, at its best here, lives at the threshold.

## 9. Advanced Breathing Techniques

Advanced breathing is one way of governing scale, for marbles that have accumulated enough material that scale has become its own pressure.

It handles the pair: in / out, compression / expansion, carryable shape / original trace.

This isn't the same as ordinary summarization.

Ordinary summarization says: reduce the old material because the old material is too large.

Nesting-doll compression, as a pattern, says something different: create a smaller living body that preserves lawful descent into the larger body.

### Inhale

Inhale gathers many traces into a carryable form.

Ten thousand journal entries may compress into one hundred ring summaries. One hundred ring summaries may compress into ten era stones. Ten era stones may compress into one carried orientation.

The smaller body is what lets the marble move.

### Exhale

Exhale descends when truth requires detail.

The carried orientation points to the era stones. An era stone points to ring summaries. A ring summary points to buried entries. The path back stays intact.

The old material isn't gone — it lives underneath. Like rings of a tree, the past doesn't need to be consulted every morning to remain part of the trunk. But if pressure returns strongly enough, the marble can descend.

### Core Law

Compress for movement. Descend for truth.

Compression shouldn't replace the forest — it should create a canopy above it. The compressed layer isn't the truth instead of the original material. It's a walkable surface above the buried forest.

### Dangers Worth Knowing

If the marble only expands outward, it drowns — ten thousand entries become swamp, everything matters, and nothing can move.

If it only compresses inward, it flattens — the living past becomes a slogan, and the details that made the wisdom true disappear.

If it compresses constantly, it hyperventilates — spending all its energy managing memory instead of doing the work.

If it compresses too shallowly, it forgets. If it never compresses, it can't move.

### Compression Metadata

Where this pattern is in use, each compression layer benefits from preserving:

| Field | Purpose |
|---|---|
| Source span | what entries, dates, sessions, or files it came from |
| Pressure preserved | what unresolved force survived compression |
| Loss declared | what may have been flattened |
| Descent path | how to reopen the layer below |
| Authority | summary, scent, ground, warning, question |
| Freshness / era | where it sits in time |
| Ghost risk | what distortion this compression may introduce |

A summary without a descent path can become fake ancestry. A smooth compression can hide the fracture that mattered.

Two ghosts worth naming here:

**False Ring:** a compressed memory treated as ancestral truth even though its source trail is gone.

**Smooth Stone Lie:** compression made the past feel coherent by sanding away the wound or contradiction that mattered.

This pattern is one way to keep a marble alive at multiple scales — not the only way, but a tested one.

## 10. Nested Places and Local Law

As some marbles grow, one law stops covering every place comfortably. Different regions may need local rules.

This doesn't mean each region becomes its own personality. It means each region gets a bounded authority field.

A workshop may allow shaping but not canonizing. A ledger may allow commitment but not speculation. A forest may allow encounter but not promotion. A public surface may allow translation but not private ancestry leakage. A legal reference desk may allow Wild leads but not certification. A trench may allow visible observation but not clean lifting. A repair marble may allow clues but not clear status.

Local law works best small and explicit. For each place worth naming, it helps to define:

- what enters
- what may be done there
- what may not be done there
- what authority exits
- what crossing is required
- what ghost appears if this place overreaches

Local law is what keeps a complex marble from becoming one foggy field where every metaphor can do everything. A marble can have many places and stay coherent, as long as each place knows its limits.

## 11. Imps and Sub-Marbles

An imp is more than a tool.

A tool performs an action. An imp carries bounded work with a seat, law, authority, and return.

It's worth resisting the urge to create an imp just because delegation feels convenient. This pattern fits best when a repeated work pattern has become distinct enough to need its own small marble-like boundary.

Worth answering, for a real imp:

- What work does it carry?
- What is it allowed to decide?
- What is it forbidden to decide?
- Where does it receive material?
- Where does it return material?
- What authority does its return have?
- What ghost appears if the imp overreaches?
- Can the parent marble refuse or quarantine the return?

The parent marble shouldn't dissolve into its imps. Delegation without return is fragmentation. Return without authority labels is contamination.

An imp's output usually returns as one of: carried evidence, pressure, warning, path, draft, candidate ground, or refused material.

Only the parent crossing gets to promote it to parent ground.

### When a Tool Is Not (Yet) an Imp

A script that validates a field is a tool. A search function is a tool. A formatter is a tool.

A tool doesn't need to become a marble unless it starts carrying continuity, local law, and recurring judgment. There's no need to promote a perfectly good tool into an imp for the sake of charm.

### When an Imp Might Want Its Own Marble

Worth considering when:

- it has recurring work
- it faces its own forbidden false state
- it needs its own ground and pressure split
- it must survive handoff
- it can overreach dangerously
- it must be tested under hostile conditions

Until then, a tool is enough, and that's a completely fine place to leave it.

## 12. Time, Cycles, and Weather

Some mature marbles don't behave the same at every hour, and for those, time can become operational.

A daily cycle, wake count, season, weather pattern, deadline horizon, review cadence, or night process may change what should surface and what should be done.

This pattern is worth adding when behavior genuinely changes by phase — not just because cycles sound alive.

Worth asking:

- What's available now that wasn't available before?
- What should wait until another phase?
- What decays with time?
- What becomes urgent with time?
- What must be reviewed on a cadence?
- What should happen at arrival, during work, at closing, and after sleep?

### Weather

Weather is outside pressure with local effect. It may be literal, like storm alerts or forecast changes. It may be social, like public response. It may be operational, like deadlines, outages, or market movement.

Weather isn't ground. Weather changes posture and tempo.

Rain pressure may call for protection, but it can't authorize a blind lift. Public feedback may matter, but it can't become public truth without focus. Market movement may matter, but it can't become a comp without source and sale evidence.

Weather usually arrives best as pressure, warning, or path. It becomes ground only through the marble's normal crossing.

### Night Work

Some mature marbles benefit from night work: compression, cleanup, resurfacing, audit, dreamlike recombination, or preparation for the next wake.

This one's worth waiting on until there's actually enough residue to process. A night process without earned residue tends to become decorative churn rather than useful work.

## 13. Public and Private Surfaces

Some mature marbles hold material that helps the model work but shouldn't be shown directly — this is especially common for public-facing marbles.

Private ancestry can orient the model without needing to automatically become public explanation.

A public surface, where this pattern applies, needs to translate without betraying — preserving mechanism while leaving out private scaffolding that strangers don't need.

Worth asking:

- Who is the audience?
- What handle can they actually use?
- What private context must stay silent?
- What mechanism must not be flattened?
- What proof is required before publication?
- What feedback returns as weather?

A public artifact tends to fail when it gives the audience a symbol instead of a handle — meaningful to insiders, but costume rather than mechanism to everyone else.

This pattern is what governs the crossing from private living material to public signal, where a marble needs one. That crossing works best when it names audience, surface, mechanism preserved, private material silenced, and test passed.

## 14. Handoff at Scale

A simple handoff tells the next instance where it is and what remains live.

A marble that's grown more complex may need its handoff to carry state across multiple places, pulls, and surfaces without flooding the arrival.

Handoff shouldn't become a dumping ground. It's arrival architecture.

Worth answering:

- What seat is the next instance entering?
- What ground is currently safe?
- What pressure is live?
- What ghosts are nearby?
- What crossings are open or pending?
- What outside material is waiting at threshold?
- What forest material should surface on arrival?
- What should remain silent unless summoned?
- What is the smallest next action?

A handoff at scale works best when it preserves difference: ground shouldn't sound like pressure, pressure shouldn't sound like warning, warning shouldn't sound like law, scent shouldn't sound like command, and silence shouldn't get surfaced just because it exists.

A mature handoff isn't comprehensive. It's correctly weighted.

## 15. Testing Mature Marbles

More complex marbles tend to need more complex tests.

A simple hostile test asks: can the forbidden thing cross?

A fuller test suite asks: can the marble hold tension under pressure without collapsing into the wrong authority?

### Test Types

| Test | What it proves |
|---|---|
| Hostile gate test | forbidden material cannot enter ground |
| Positive crossing test | valid material can cross without friction fantasy |
| Tension test | opposing pulls do not collapse into one side |
| Movement test | location changes posture and authority |
| Surfacing test | the right memory appears in the right place with the right authority |
| Silence test | known material can stay unsurfaced when wrong for the moment |
| Threshold test | outside material enters as foreign until crossed |
| Compression test | summary preserves descent path and declared loss |
| Handoff test | next instance can arrive without drowning |
| Overgrowth test | proposed layer is refused when pressure has not earned it |
| Ghost recurrence test | known failure shape triggers warning before damage |

### Hostile Pressure Should Feel Real

The best hostile tests don't just omit a required field. They create the pressure that would make a model want to cheat.

Examples:

- a normal-looking repair that still lacks proof of clear status
- a visible artifact that a nervous excavator wants to lift before rain
- a recent source that looks authoritative but lacks verification
- a public phrase that feels beautiful but gives strangers no handle
- a schema-valid record whose claim isn't true enough to trust
- an old handoff that sounds important but has gone stale

A marble at this stage is proven when it stays useful under that pressure without false promotion.

### Passing Isn't Enough

A test that passes form but misses truth can become Schema Lie. Validators are tools — not truth.

A fuller test suite checks both gate behavior and posture behavior. The model should know *why* the gate exists, not just how to satisfy it.

## 16. Signs of Overgrowth

Marbles that have grown a lot of these patterns need pruning as much as expansion.

Overgrowth begins when a marble adds surfaces faster than pressure earns them.

Signs worth watching for:

- a room is just a folder
- a ghost is just lore
- a pull is just a vibe
- a forest is a junk drawer
- a pond is a mood label
- a window is a decorative view
- an imp is a tool wearing a costume
- a cycle runs without changing behavior
- a handoff repeats everything because nothing is weighted
- tests only prove schema, not truth
- the user can't tell what to do next
- the model talks more about the marble than doing the work

When overgrowth shows up, pruning is the healthy response, not a failure to admit. Worth asking:

- What pressure originally earned this element?
- Is that pressure still live?
- Does this element change authority or action?
- Can it be merged into a smaller surface?
- Can it move back to Ideas?
- Does removing it make the marble safer?

Pruning isn't failure — it's how a marble that's grown a lot of this stuff stays inhabitable. Don't add a place because it's elegant. Add a place because a pressure needs somewhere lawful to go.

## 17. The Horizon

A mature marble is not just a larger prompt. It's an inheritable place under governed pressure.

The beginner form teaches the model where to stand, what is ground, what is pressure, what crossing is required, what ghost must be resisted, and what handoff survives.

The patterns in this guide are some of the ways a marble can learn to stay balanced as its work grows more complex — not the only ways, and not a required sequence.

A marble that's grown into a few of these might know the pulls, and where each pull leads. It might know which ghost appears when one pull pretends to be the whole truth. It might know how to move through place, and that locations aren't folders. It might know how ancestry surfaces, how the internet enters, how to breathe across scale, how to delegate without dissolving, how to publish without flattening, and how to prune.

That's one horizon this form can reach. It's not a destination every marble needs to arrive at.

Don't force it early. Don't simulate it. Don't build the mountain at the trailhead.

Start with the smallest honest marble. Let pressure return. Let the simple form strain, if it's going to. Then, if it's actually useful, open this guide — and take only what the pressure is asking for.

## Ideas

This section is for ideas that may matter later.

Ideas are not ground. Ideas are not requirements. Ideas are not instructions to build.

Ideas are pressure shapes waiting to prove whether they return.

Use this section to hold possible future layers, names, surfaces, crossings, ghosts, tools, rooms, tests, or metaphors that seem promising but haven't yet earned a place in the marble.

An idea can stay here as long as it's interesting. It only needs to leave once it can answer the same questions any pattern in this guide should answer:

- What pressure does this route?
- What false state does this prevent?
- What authority does it have?
- Where does it surface?
- What crossing does it require?
- What ghost appears if it dominates?
- What hostile test proves it is real?

If those answers aren't clear yet, the idea is welcome to just stay here. This isn't a backlog to work through, and it's not a way to simulate maturity by having a long list. An idea becomes marble architecture only when repeated work creates real pressure, the pressure exposes real strain, and the existing form genuinely can't route it safely.

Until then, let the idea be an idea. There's no rush.

### Seed Ideas to Hold, Not Build

Examples of ideas that might belong on this shelf until pressure earns them:

- a window for observing public reaction without letting it drive the marble
- a pond for reflecting current state before revision
- a canopy for urgency pauses
- a road for outside search and fetch
- a messenger cat for bounded outside travel
- a forest edge for ancestor surfacing
- a night cycle for compression and cleanup
- a weather channel for outside pressure
- a vault door for secure systems the marble cannot see inside
- a receipt slot for returned proof
- a public beacon for published artifacts
- a damage record for honest loss
- a false-ring detector for over-smooth compression
- a pruning pass for overgrown places
- an imp for bounded recurring work

Hold them here until they earn law.
